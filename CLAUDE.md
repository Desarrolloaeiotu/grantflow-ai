## 1. Identidad del proyecto

**Nombre:** GrantFlow AI  
**Organización:** aeioTU — Gerencia de Alianzas Internacionales  
**Propósito:** Sistema de inteligencia comercial para prospección estratégica de oportunidades de financiamiento. Transforma la búsqueda manual de grants en un proceso automatizado, estandarizado y escalable.  
**Posición en el proceso:** Opera ANTES del CRM. Entrega oportunidades ya evaluadas y priorizadas. No reemplaza el CRM — lo alimenta con calidad.

**Problema que resuelve:**
- Búsqueda manual dispersa en múltiples fuentes no integradas
- Evaluación inconsistente de oportunidades (depende del criterio individual)
- Sin trazabilidad del pipeline explorado, descartado o en análisis

**Meta 2026 que habilita:**
- Pipeline activo ≥ COP $18.000M (≥5× la meta anual)
- ≥3 oportunidades ≥$400M por trimestre
- 8–10 oportunidades estratégicas en gestión permanente
- Tasa de conversión ≈25%
- ≥85% de metas comprometidas antes del 30 de noviembre

### Reglas de validación de email (Apollo.io)

```
REGLA: Ningún email (org_email, ceo_email) se almacena sin pasar por Apollo.io.

Estados posibles del campo *_email_verified:
  TRUE  → Apollo confirma que el email existe y acepta correo (confidence ≥ "high")
  FALSE → No verificado aún O Apollo lo marcó como inválido/risky

Flujo obligatorio:
  1. Scraper detecta oportunidad → guarda org_email / ceo_email como NULL
  2. n8n trigger "enrich-contacts" → llama Apollo /email/verify
  3. Si verified=true  → actualiza *_email_verified=TRUE, *_email_verified_at=NOW()
  4. Si verified=false → mantiene email pero marca verified=FALSE y loggea en score_log
  5. Emails NO verificados aparecen en UI con ícono ⚠️ — NUNCA se usan en campañas automáticas

Apollo.io endpoint de verificación:
  POST https://api.apollo.io/api/v1/people/match
  Body: { "email": "ceo@org.com", "reveal_personal_emails": false }
  Campo clave de respuesta: person.email_status ("verified"|"likely"|"unknown"|"invalid")
  Guardar como verified=TRUE solo si email_status == "verified"
```

## 5. Lógica de negocio crítica

### 5.1 Motor de scoring — Los 5 criterios de aeioTU

**REGLA DE ORO: Una oportunidad es Go si score_total ≥ 6 (de 10 posibles).**  
Cada criterio vale máximo 2 puntos. El LLM evalúa criterios cualitativos; las reglas evalúan cuantitativos.

```python
# backend/app/services/scoring_engine.py

SCORING_RULES = {
    "criterion_1_alignment": {
        "name": "Alineación estratégica",
        "description": "El financiador apoya lo que aeioTU ya implementa o busca consolidar",
        "method": "llm",  # Claude evalúa semántica
        "keywords": ["early childhood", "ECD", "educación inicial", "primera infancia",
                     "docentes", "formación", "política pública", "escalamiento"]
    },
    "criterion_2_model": {
        "name": "Ajuste del modelo",
        "description": "El financiador prioriza modelos escalables y sostenibles",
        "method": "llm",
        "keywords": ["scalable", "sustainable", "replication", "transfer", "model",
                     "escalable", "sostenible", "transferencia", "replicable"]
    },
    "criterion_3_ticket": {
        "name": "Coherencia del ticket",
        "description": "El monto permite estructurar una intervención viable",
        "method": "rule",
        # Rango viable aeioTU: COP $400M – $5.000M
        "min_cop": 400_000_000,
        "max_cop": 5_000_000_000,
        "points": {
            2: "within_range",      # Dentro del rango ideal
            1: "adjacent_range",    # ±30% del rango
            0: "out_of_range"       # Fuera del rango
        }
```

## 10. Flujos n8n

### Flujo 1: `daily-scraping`
- **Trigger:** Cron `0 6 * * *` (6am diario)
- **Pasos:** HTTP Request → cada Azure Function/endpoint de scraper → Supabase: upsert oportunidades → HTTP POST scoring automático para nuevas oportunidades
- **Error handling:** Si falla un scraper, continuar con los demás. Notificar a Slack `#dev-alerts`.

### Flujo 2: `deadline-alerts`
- **Trigger:** Cron `0 9 * * *` (9am diario)
- **Lógica:** Consultar opportunities WHERE deadline = NOW() + 7 OR +15 OR +30 días AND decision = 'go'
- **Output:** Adaptive card a Slack `#alianzas` + email vía Resend

### Flujo 3: `weekly-digest`
- **Trigger:** Cron `0 8 * * 1` (lunes 8am)
- **Lógica:** Top 10 oportunidades Go ordenadas por score DESC, semana de detección
- **Output:** Email HTML formateado a equipo de Alianzas
