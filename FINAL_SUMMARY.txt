BACKOFF LOGIC VISUALIZATION
───────────────────────────────────────────────────────────────────────────────

Retry Sequence (if all attempts fail):

Attempt 1  ─→ GET (timeout or 403)
           ↓
           [Wait 1 second]  ← backoff = 2^0 = 1
           ↓
Attempt 2  ─→ GET (timeout or 403)
           ↓
           [Wait 2 seconds]  ← backoff = 2^1 = 2
           ↓
Attempt 3  ─→ GET (timeout or 403)
           ↓
           [Wait 4 seconds]  ← backoff = 2^2 = 4
           ↓
Attempt 4  ─→ GET (timeout or 403)
           ↓
           ❌ FAILED (total time: ~7.5 seconds)

Total wait time = 1 + 2 + 4 = 7 seconds
Plus request overhead ≈ 0.5 seconds
= ~7.5 seconds maximum per request

───────────────────────────────────────────────────────────────────────────────
HEADERS ROTATION STRATEGY
───────────────────────────────────────────────────────────────────────────────

Each request gets random headers to avoid pattern detection:

USER-AGENT POOL (6 options):
  1. Chrome Windows
  2. Chrome macOS
  3. Chrome Linux
  4. Firefox Windows
  5. Safari macOS
  6. Edge Windows

REFERER POOL (4 options):
  1. Google
  2. Bing
  3. Yahoo
  4. Grants.gov direct

Result: 6 × 4 = 24 unique header combinations
Probability of same combo twice: 1/24 ≈ 4%

───────────────────────────────────────────────────────────────────────────────
RATE LIMITING STRATEGY
───────────────────────────────────────────────────────────────────────────────

Between pages:        0.5 seconds (reasonable pacing)
Between search terms: 1.0 seconds (courteous to server)
Between retries:      Exponential (1, 2, 4 seconds)

Example: 7 search terms × 3 pages each
├─ Without failures: ~3 requests/sec × 21 = ~7 seconds
├─ With 1 failure per page: +7 seconds backoff = ~14 seconds
└─ With all failures: +49 seconds backoff = ~56 seconds

Doesn't block main application (async non-blocking)
